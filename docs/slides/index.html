A field guide to an S3-native search engine

Zeppelin

Object storage is the source of truth. Nodes are stateless.

✦

This primer explains how Zeppelin — a vector and full-text search engine written in Rust — stores everything it knows in an S3 bucket, coordinates concurrent writers without a lock server, and answers nearest-neighbor queries in ~100 ms.

It is written for a technically competent reader who is not a database specialist. Every piece of jargon is defined the first time it appears, every claim links to the code that implements it, and every borrowed idea links to the paper or engineering post it came from.
